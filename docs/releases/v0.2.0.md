# pulse-kestra v0.2.0 リリースノート

日付: 2026-03-15

## 概要

Phase 2 の回復運用を実装し、`agent-taskstate` を正本にした durable dedupe、heartbeat 巡回、manual replay、notifier resend を実運用契約に揃えました。あわせて `reply_text` と `roadmap_request_json` を task 正本へ保存し、worker 再実行なしの通知再送と trace ベースの replay を成立させています。

## 主要な変更

- bridge 起票時に `reply_target` `trigger` `roadmap_request_json` を task に保存
- mention flow で `reply_text` を task 正本へ保存し、reply 成否に応じて `reply_state` を更新
- manual replay flow で保存済み `roadmap_request_json` と `reply_text` を再利用
- notifier-resend flow を task 正本ベースの再送導線へ整理し、失敗時は `retry_count` を更新
- heartbeat flow を回復導線の起動役へ整理し、`--reply-state` と `--updated-before` を前提に notifier-resend / manual-replay を起動
- Phase 2 の taskstate 契約を前提に bridge test / Kestra flow contract test を補強
- `pytest-asyncio` の deprecation warning を解消する設定を追加

## 検証結果

- `python -m pytest -q` 実行結果: 151 passed
- deprecation warning: 0
- `trace_id` `reply_text` `roadmap_request_json` を含む replay / resend 契約をテストで固定
